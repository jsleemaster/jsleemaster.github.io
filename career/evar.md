## 📋 업무 요약

- 레거시 아키텍처(MVVM, GetX)를 **Clean Architecture와 Riverpod로 성공적으로 전환**하여, 앱의 확장성과 테스트 용이성을 확보하고 API 캐싱을 통해 서버 통신을 최적화했습니다.

- **네이버 맵 SDK**를 활용하여 지도 기능을 개발 진행하였으며, 뷰포트 기반 로딩, 마커 캐싱 등 최적화 기법을 적용하여 사용자 경험을 크게 개선했습니다.

- **iOS 다이나믹 아일랜드**를 도입하여 사용자 편의성을 극대화했으며, **Github Actions, Fastlane, Xcode Cloud**를 연동하여 AOS/IOS 자동 배포 파이프라인을 구축, 개발 생산성과 배포 안정성을 크게 향상시켰습니다.

<br/>

## 🪴 경험 및 배운점

### '지금 만드는 사람'보다 '미래의 우리'를 위한 선택을 고민하게 되었습니다.

EVPAY를 개발할 당시, Flutter 네이티브 위젯으로 구현하는 것이 가장 빠르고 직관적인 방법이었다고 생각하였습니다. 하지만 서비스가 성장하면서 정책과 UI 변경 요청이 잦아졌고, 그때마다 앱 심사를 다시 받아야 하는 비효율이 발생했습니다.

배포 후 든 생각은 **'웹뷰로 만들었다면 버튼 하나 바꾸는 데 하루가 아니라 10분이면 됐을 텐데'**라는 생각이 들었습니다. 

이 경험을 통해 개발 결정은 지금 당장의 공수뿐만 아니라, **향후 운영까지 고려해야 한다는 것**을 느꼈습니다. 이 기능의 '변경 주기'와 '중요도'를 먼저 생각하고 가장 효율적인 기술 스택을 선택하는 습관을 갖게 되었습니다.
